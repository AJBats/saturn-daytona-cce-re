## Important findings from this session worth carrying forward

### Workflow discoveries (in `dead_code_sweep_protocol.md`)

- **Mednafen WP doesn't fire on DMA writes** (race.bin loads via SCU DMA). The workaround is a value-filtered WP at the first 4-byte word that DIFFERS between SLCT.BIN and race.bin (currently 0x06028008, value 0x2FB6EC00). The CPU patches that word during init's loader → the WP catches the patch → race.bin is loaded.
- **`sample_memory` advances at 60fps display rate**, not Daytona's 30fps logic rate. So 1 minute of capture = 3,600 mednafen frames, not 1,800.
- **Arcade mode times out** if the poke replay doesn't drive through checkpoints. CS3+CS4 worked because the captured laps include checkpoint-passing positions; CS2 first attempt (arcade) timed out and reloaded SLCT, polluting the BP set. TT mode has no timeout.
- **Oneshot BP mode (the new mednafen feature)** is the difference between 14 GB logs at 1 fps and 600 KB logs at 60 fps. Use it for any 1000+ BP sweep.

### Scenario gaps we haven't covered

These could fire functions our sweep didn't see:
- **Real controller input** — throttle/brake/steer/gear shifts. Most of the input chain is NOPped in transplant so the impact is bounded, but not zero.
- **Crashes** — wall hits, off-track, car-to-car. Collision response is NOPped at the dispatch level but downstream code might still fire briefly.
- **Race-end / results** — finish line, position display, score. Not exercised in our sweeps.
- **Pause / retry** — pause-handler, retry-from-results. Not exercised.
- **Other modes** — championship, 2P, free run.
- **Arcade timer logic** — the very thing that timed us out in CS2's first arcade attempt is itself a function set we'd need to verify isn't in our delete list. User's idea: RE the timer + freeze it for arcade-coverage experiments.

### Big design realizations

- **The decomp `make disc` target was opaquely mod-specific** — fixed 2026-04-29. Now `make -C decomp disc` (no MOD) → pristine, `make -C decomp MOD=transplant disc` → transplant. Mirrors parent convention.
- **The 42 KB DUSA estimate was a hand-wave** — actual budget per DUSA's `transplant_manifest.md` is ~27-28 KB worst case (player + AI + math + tables). Math savings (12 KB) possible if CCE math equivalents hold (deferred audit).
- **The "river" alone is too small** to fit DUSA — only 8.4 KB strict driving-model dead. We need to draw on the broader CDL_NOT_EXECUTED bucket (53 KB) for room. That broader bucket isn't river-traced but IS empirically validated by both sweep and CDL.
