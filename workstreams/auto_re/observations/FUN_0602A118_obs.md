## Behavior

FUN_0602A118 is the **segment activation/marking function**. It reads
BLK segment pointer tables (+0x24 in BLK structure), traverses linked
segment lists with 0x30 stride, and marks each found segment with a
byte flag (value 1) in an activation buffer.

This is the core mechanism for track cell streaming — it determines
which segments are "active" (visible/loaded) based on the car's current
position. Downstream code reads the activation buffer to decide which
track geometry to render.

Calls FUN_0602A2FC for individual segment data access and
FUN_0602A2A4 / indirect calls for segment processing.

## Transplant Implications

Must remain functional — this is the segment activation system that
rendering depends on. Reads BLK data to determine visibility. Works
with any valid car position in world coordinates.
