## Overview

FUN_06035904 is called as sub-function #12 from the dispatcher (FUN_0604D380).
Large function using the same constants as FUN_060354A0 (0x03700000, 0x02D00000)
in cross-multiply chains with values from +0xE0-EC. Contains 3 helper
sub-functions in its TU: FUN_06035B0E, FUN_06035B30, FUN_06035C58.

The final output is written to +0xF0 after 8 arithmetic right shifts (>> 8),
and +0xA0 is zeroed/set as a secondary output.

## Computation Pattern

1. Read 32-bit values from +0xE0/+0xE4 and +0xE8/+0xEC (all negated)
2. Cross-multiply: (+0xE8 * +0xE4 * 0x03700000) XOR-gated with (+0xEC * +0xE0 * 0x02D00000)
3. Write intermediate to +0xFC
4. Compare +0xC8 vs +0xDC, compute delta, blend with +0xFC result (shar x2)
5. If magnitude > +0xF4: call FUN_0604818C (integer sqrt), write result to +0x6C
6. Similar magnitude computation, sqrt, write to +0x70
7. Conditional call to FUN_06035B0E (helper: timer at +0x178, gated by +0x34)
8. Compute -(+0xAC) >> 3 (arithmetic shift right 3), write to +0x14 as 16-bit
9. Flag-gated block (+0x190): zero +0x64/+0x68 and +0xE8/+0xEC
10. Flag-gated block (+0x170): zero +0x14, +0x64/+0x68, +0xE8/+0xEC, set +0x100
11. Call FUN_06035B30 (sub: sqrt, table lookup at +0x1B2, clamping)
12. Final computation: combine +0x6C * stack_val + stack_val2 * +0x70 - +0xBC,
    multiply by constant (0x89 or 0x72 depending on +0x1C7 byte), shift right 8
13. Write final value to +0xF0 (rts delay slot)

## Other Observations

- This function uses the same constants (0x03700000, 0x02D00000) as FUN_060354A0
  but in a different computation structure (cross-product vs rotation transform).
- The +0x1C7 byte appears in both this function and FUN_060354A0. In FUN_060354A0
  it selects shift amount (>> 2 vs >> 4). Here it selects between multipliers
  0x89 (137) and 0x72 (114) — a ~20% difference.
- +0x14 is WRITTEN by this function (derived from +0xAC via negation and >> 3).
  It is not a direct input from elsewhere in this context.
- The zeroing blocks (gated by +0x190 and +0x170 flags) reset multiple fields
  simultaneously: +0x64, +0x68, +0xE8, +0xEC, and optionally +0x14 and +0x100.
