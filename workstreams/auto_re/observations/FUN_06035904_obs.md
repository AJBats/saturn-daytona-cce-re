## Overview

FUN_06035904 is called as sub-function #12 from the player physics dispatcher
(FUN_0604D380). It is a large physics function that computes lateral/longitudinal
force components using sin/cos heading data and the same friction constants as
FUN_060354A0 (0x03700000, 0x02D00000). Contains 3 helper sub-functions in its TU:
FUN_06035B0E, FUN_06035B30, FUN_06035C58.

The final output is written to +0xF0 after 8 arithmetic right shifts (÷256),
and +0xA0 is zeroed/set as a secondary output.

## Computation Pattern

1. Read sin/cos heading components from +0xE0/+0xE4 and +0xE8/+0xEC (all negated)
2. Cross-multiply: (+0xE8 * +0xE4 * 0x03700000) XOR-gated with (+0xEC * +0xE0 * 0x02D00000)
3. Write intermediate to +0xFC
4. Compare +0xC8 vs +0xDC, compute delta, blend with +0xFC result (shar²)
5. If magnitude > +0xF4: call FUN_0604818C (sqrt), write result to +0x6C
6. Similar magnitude computation → sqrt → write to +0x70
7. Conditional call to FUN_06035B0E (helper: timer at +0x178, speed-gated)
8. Derive steering: negate +0xAC, shift right 3, write to +0x14 as 16-bit
9. Flag-gated block (+0x190): zero +0x64/+0x68 and +0xE8/+0xEC
10. Flag-gated block (+0x170): zero +0x14, +0x64/+0x68, +0xE8/+0xEC, set +0x100
11. Call FUN_06035B30 (complex sub: sqrt, table lookup, velocity clamping)
12. Final computation: combine +0x6C * stack_val + stack_val2 * +0x70 - +0xBC,
    multiply by constant (0x89 or 0x72 depending on +0x1C7 flag), shift right 8
13. Write final value to +0xF0 (rts delay slot)

## Helper Sub-Functions

### FUN_06035B0E (timer helper)
- Reads +0x178 (16-bit timer), +0x34 (speed)
- If timer <= 4 AND speed >= 20: writes 0x0F (or 0x07 if timer nonzero) to +0x178
- Simple counter/timer management

### FUN_06035B30 (velocity clamping sub)
- Complex: reads speed, +0xD0, +0xC4, calls sqrt twice
- Uses table at offset 0x1B2 from GBR for lookup
- Conditionally calls FUN_06035C58 and writes to +0x144, +0x100
- Clamps +0x70 to [0x1999, 0x10000] and adjusts +0x100

### FUN_06035C58 (flag/timer sub)
- Reads +0x176 (timer), speed
- Writes to +0x176, sets bit 0x2000 in @(48, r0) = +0x30

## Other Observations

- This function shares the same friction constants as FUN_060354A0 but computes
  different force components — likely the lateral force counterpart.
- The +0x1C7 byte flag appears in both this function and FUN_060354A0. In
  FUN_060354A0 it gates damping rate (÷4 vs ÷16). Here it selects between
  force multipliers 0x89 (137) and 0x72 (114) — a ~20% difference.
- The steering write to +0x14 is notable: this function DERIVES steering from
  +0xAC (negated, shifted right 3). So +0x14 is not raw controller input —
  it's a processed steering value.
- The mass zeroing blocks (gated by +0x190 and +0x170 flags) suggest state
  transitions that reset multiple physics channels simultaneously.
