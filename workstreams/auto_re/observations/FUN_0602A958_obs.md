## Memory Writes

Calls FUN_06047E0C (atan2) with position deltas from R10-based struct.
Writes results to:
- R10 + DAT_0602a9d8 offset: computed heading (16-bit)
- DAT_0602a9dc: global heading value
- R10 + DAT_0602a9e0 offset: direction sector (8 sectors, from heading delta)

## Behavior

FUN_0602A958 is the **direction-of-travel detector**. It:

1. Calls atan2 (FUN_06047E0C) on BLK segment direction vectors to get
   the track's heading at the car's position
2. Compares track heading with car heading (R10 + 0x0E)
3. Computes a direction sector (0-7) from the heading difference
4. Sets a direction flag: 1 for forward, 0xFF for reverse

This is the function that triggers the "REVERSE" warning in the HUD
when the player drives backwards. The NOP test of FUN_06029D8C (its
caller) showed incorrect reverse detection — confirming this function's
role.

## Transplant Implications

The direction detector reads BLK direction vectors and car heading.
For the transplant:
- If BLK data is preserved (coexistence model), this works automatically
- DUSA physics must write a heading value that this function can compare
  against BLK track heading (heading convention must match or be adapted)
