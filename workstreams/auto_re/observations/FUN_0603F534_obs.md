## Gate Mechanism

FUN_0603F534 is gated by GBR+194 (byte):
- If GBR[194]==0: immediate return (no-op)
- If GBR[194]!=0: falls through to FUN_0603F53E (the active path)

GBR+194 is set by FUN_0603F58C (called from FUN_0603EF86 at PR=0x0603EF8C).
F58C writes GBR+194 to a nonzero countdown value and GBR+195 to a threshold.

## Value Dynamics

### Field: GBR+194 (countdown)

Entry 0 countdown sequence (from watchpoint run):
- Frame 779: set to 0x09 (by FUN_0603F58C)
- Frame 785: 0x07 (decremented by 2 in 6 render frames)
- Frame 800: 0x00 (countdown complete)

The countdown decrements by 1 each physics frame. Physics runs approximately
every 3 render frames. Over 9 physics frames (~27 render frames), the
countdown goes from 9 to 0.

### Field: GBR+26 (lateral correction)

- During countdown: changes to nonzero values (e.g., 0xFE00 at frame 785)
- When countdown reaches 0: reset to 0x0000

The final `mov.w r0, @(26, gbr)` at line 720 (.L_0603F572) writes the result
of the drift calculation. When `dt r0` produces zero (countdown done), the
branch at line 694 (`bt .L_0603F572`) jumps to the write with r0=0, resetting
GBR+26 to 0.

## Other Observations

- FUN_0603F534 is almost always a no-op (2 instructions: gate check + rts).
  The active path (FUN_0603F53E) executes only during a brief countdown
  window (~9 physics frames) triggered by FUN_0603F58C.

- The Verifier's "ALL FAIL" result is explained: GBR[194]=0 for all cars
  at the test frames. The function cannot produce observable writes during
  the standard race_idle test window (frames 0-120).

- FUN_0603F58C (the trigger) is called from FUN_0603EF86 (in FUN_0603EC54.s),
  which is part of a larger pipeline function. F58C computes the drift
  parameters using the hardware divider and speed (GBR+72).

- The PC trace at frame 390 shows 5 hits at both 0x0603F53A (rts, no-op
  path) AND 0x0603F53E (FUN_0603F53E entry). The F53E hits with zero hits
  at F53E+2 (0x0603F540) suggest these are PC trace branch-resolution
  artifacts rather than actual execution of F53E.

- Physics dispatch does not run every render frame. Approximately every
  3 render frames at frame ~785 (2 physics ticks in 6 render frames).
  This affects countdown duration: 9 physics frames ≈ 27 render frames.

- The activation frame for entry 0 varied between runs: frame 779 (with
  prior watchpoint history) and frame 1185 (clean run). This variability
  suggests the trigger condition depends on car state that evolves
  nondeterministically across runs.

- GBR+195 is read by F534 at line 696 (`mov.b @(195, gbr), r0`) and used
  in a comparison (`cmp/ge r0, r1`). It controls a threshold for the drift
  calculation. Set to 0x05 when GBR+194 is set to 0x09.
