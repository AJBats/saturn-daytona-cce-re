## Overview

FUN_060354A0 is called as sub-function #10 from the player physics dispatcher
(FUN_0604D380). It is a force/velocity integration function that computes
X/Z position deltas from heading, velocity, and engine force data.

Contains massive DMULS (64-bit multiply) chains for fixed-point physics
calculations. Calls the same trig function (FUN_06047D20) as FUN_0604DD04.

Helper function FUN_06035624 (at 0x06035624) is called twice via BSR to
process paired X/Z components.

## Computation Pattern

1. Read two pairs of velocity vectors (+0x4C/+0x50, +0x54/+0x58)
2. Call FUN_06035624 helper for each pair — applies speed gate check (@+0x34 >= 10?)
3. Read sin/cos components from +0xE0/+0xE4 and +0xE8/+0xEC (negated)
4. Apply heading rotation: multiply velocity by heading direction
5. Multiply by friction coefficients (0x03700000 ≈ 0.215, 0x02D00000 ≈ 0.176)
6. Call FUN_06047D20 with heading angle — get sin component
7. Multiply result by 0x537CE965 (large constant, traction model)
8. Accumulate forces
9. Clamp X force to [0x015FFFE0, 0x0C058000] (asymmetric — forward bias?)
10. Clamp Z force to [0x011FFFF8, 0x09D58000]
11. Read +0xF4/+0xF8 (old position), compute delta, apply damping
12. **Damping gate**: read +0x1CB (byte). If value == 2: shift right 2 (÷4).
    Otherwise: shift right 4 (÷16). Higher grip = less damping = faster response.
13. Add delta to position, write back to +0xF4/+0xF8

## Other Observations

- The damping gate at +0x1CB is very interesting — a single byte controls whether
  force changes are divided by 4 or 16, a 4x difference in response rate. This
  could be surface type (asphalt vs grass vs sand) or tire compound selection.
- The function pair (+0xF4, +0xF8) was seen changing dramatically between scenarios
  in the GBR field survey: idle stays constant, throttle/steer produce large changes.
  This is consistent with being the primary force/position accumulator.
- FUN_06035624 helper checks `@(52, r0)` which is GBR+0x34 (displayed speed).
  If speed < 10 km/h, it takes a different code path (early return to FUN_06035748).
  This suggests low-speed physics has special handling — possibly slip angle
  calculation is bypassed below walking speed.
- The asymmetric clamp values (X range wider than Z range) suggest the physics
  model handles lateral vs longitudinal forces differently.
- From mem_sample data (300 frames, throttle held): +0xF4 and +0xF8 are among
  the most actively changing fields (143 unique values each over 300 frames).
  Both are classified as "changing" in the full struct analysis.
