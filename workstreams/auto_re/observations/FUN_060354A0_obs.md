## Overview

FUN_060354A0 is called as sub-function #10 from the dispatcher
(FUN_0604D380). Contains DMULS (64-bit multiply) chains with fixed-point
arithmetic. Calls FUN_06047D20 (same trig function used by FUN_0604DD04).
Reads from +0xE0-EC, +0xF0, +0xF4, +0xF8 and writes back to +0xF4/+0xF8.

Helper function FUN_06035624 (at 0x06035624) is called twice via BSR to
process paired components.

## Computation Pattern

1. Read two pairs of 32-bit values from +0x4C/+0x50 and +0x54/+0x58
2. Call FUN_06035624 helper for each pair — gates on @+0x34 >= 10
3. Read 32-bit values from +0xE0/+0xE4 and +0xE8/+0xEC (negated)
4. DMULS chain: multiply pair values by +0xE0-EC values (rotation transform)
5. Multiply by constants 0x03700000 (~0.215) and 0x02D00000 (~0.176)
6. Call FUN_06047D20 with +0x0C (16-bit value) as argument
7. Multiply result by 0x537CE965
8. Accumulate into intermediate
9. Clamp result A to [0x015FFFE0, 0x0C058000] (asymmetric range)
10. Clamp result B to [0x011FFFF8, 0x09D58000]
11. Read +0xF4/+0xF8 (previous values), compute delta, apply shift
12. **Conditional shift**: read +0x1CB (byte). If value == 2: shift right 2 (divide by 4).
    Otherwise: shift right 4 (divide by 16). The byte value selects between two rates.
13. Add delta to +0xF4/+0xF8, write back

## Other Observations

- The conditional shift at +0x1CB selects between dividing by 4 and dividing by 16,
  a 4x difference in the rate at which the delta is applied to +0xF4/+0xF8.
- +0xF4 and +0xF8 are static at idle but among the most actively changing fields
  with B held (143 unique values each over 300 frames in the throttle capture).
- FUN_06035624 helper checks `@(52, r0)` which is GBR+0x34.
  If +0x34 < 10, it takes a different code path (early return to FUN_06035748).
  Below this threshold, the main computation is skipped.
- The clamp ranges are asymmetric: result A range [0x015FFFE0, 0x0C058000] is
  wider than result B range [0x011FFFF8, 0x09D58000].
- +0xE8 and +0xEC both decrease monotonically from nonzero values toward 0 even
  at idle (112 and 110 unique values respectively over 300 frames). With steering
  input, they fluctuate (150 unique values each).
