## Function Purpose (from behavior, not interpretation)

FUN_060386D8 is the **terrain surface processor**. It:
1. Calls FUN_06036AA8 three times per frame (from PCs 0x0603874A, 0x06038798,
   0x060387E6 — confirmed from FUN_06036AA8_obs.md)
2. Each call performs a spatial grid lookup at a different position (likely
   the car's current position and two neighboring points for interpolation)
3. Reads surface properties from the polygon table entries
4. Writes terrain-derived fields to the car struct:
   - **+0x04**: Y position (terrain height)
   - **+0x10**: Banking/curvature angle (zero on flat, negative on right curves)

## Other Observations

- The function calls FUN_06036AA8 THREE times, using the car's X/Z position
  and two offset positions. This is likely a 3-point terrain sampling for
  interpolated height and normal vector computation.
- Both +0x04 and +0x10 are externally computed from the track geometry data.
  The driving model does NOT produce these values — they are derived from
  the car's XZ position via the spatial grid → polygon → surface property
  pipeline. This is critical for the transplant: the '95 model only needs
  to produce X and Z, and +0x04 (Y) and +0x10 (banking) will be computed
  automatically by CCE's terrain system.
- +0x10 banking values on Three Seven: 0 (flat straight) to 0xF5200000
  (peak banking on the right curve). The value is a 16.16 fixed-point angle
  with the upper 16 bits carrying the signed angle.
