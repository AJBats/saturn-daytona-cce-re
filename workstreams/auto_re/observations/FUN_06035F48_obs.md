## Gate Conditions (from static analysis, confirmed by watchpoint absence)

The function has a heavily gated setup section. ALL conditions must be met for the
main write path to activate:

1. `r14[0x176]` (word) <= 0 — if > 0, skip setup entirely
2. `r14[0x34]` >= 100 (0x64) — speed-like threshold
3. `r14[0x14]` (word) != 0 — nonzero check
4. `r14[0xF8]` <= `GBR[0x148]` — cross-field comparison
5. `r14[0x68]` in range [-0x300, +0x300] — bounded magnitude check
6. `r14[0x44]` in range [-0x1B0, +0x1B0] — bounded magnitude check

When the setup fires, it sets +0x18E (a countdown timer) and OR's +0x30 with 0x02.
The +0x18E timer then enables the +0x70 write path on subsequent frames.

In idle and pure throttle, condition 2 (+0x34 < 100) blocks everything.
In steer+throttle from standing start, +0x34 never reaches 100 (wall strike at
~frame 140 interrupts speed buildup), so the setup never fires either. The writes
to +0x64, +0x68, +0x104 happen on a separate unconditional code path AFTER the
gated section.

## Other Observations

- +0x64 starts changing earlier than +0x68 in steer+throttle: +0x64 is already at
  0x1FE by frame 10, while +0x68 stays at 0x01 until frame 30. +0x64 leads +0x68.
- Both +0x64, +0x68, and +0x44 simultaneously reset to 0 at the wall strike (~frame 140).
  This is a clear collision response event. The fields recover after the strike.
- +0x68 recovers more slowly than +0x64 after the wall strike (still 0 at frame 160,
  while +0x64 is already 0x7F0).
- The function calls FUN_06047D3C (sin/cos), FUN_06047E0C (atan2), and DAT_06048160
  (math helper) multiple times — heavy trigonometric computation.
- The function calls FUN_06035C58 via BSR (line 526-527 in assembly) — this is a
  subroutine in the same TU, before FUN_06035C98.
- +0x44 is loaded into r10 at function entry and used throughout as a steering angle
  input. When +0x44 = 0 (no steer), all computations produce zero output regardless
  of other field values.
- The two call sites (PR=0x0604D43E and PR=0x0604D44A) may process different axes
  or components — the dispatcher calls this function twice in sequence, possibly with
  different parameters set up by preceding sub-functions.
- +0x34 in steer+throttle only reaches 0x40 (64 decimal) at frame 299, well below the
  gate threshold of 100. In pure throttle it reaches 0x92 (146) at frame 299 — steering
  slows the speed buildup. This means the gated +0x18E timer setup path requires either
  a longer run or a higher initial speed than our save state provides.
