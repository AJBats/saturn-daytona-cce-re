## GBR Fields Accessed (Static Analysis)

### FUN_0604D580 reads:
- +0x78: Raw steering angle (4 bytes, signed). 0 when no steer input, 0x69 with RIGHT.
- +0x0C: Reference angle (wpool offset 0x0C). Constant 0x3FFF (heading).
- +0x50: Clamp range (wpool offset 0x50). Constant 0x00000001.

### FUN_0604D580 writes:
- +0x7C: Processed steering output (clamped, scaled from +0x78)
- +0x88: Input state value (via clamp to 0x38-0xB8 range)
- +0x8C: Scaled input value (via FUN_06048160 math helper)
- +0x98: Second input state value (clamped 0x38-0xB8 range)
- +0x9C: Second scaled input value

### 0x0604D6B8 reads:
- +0x24: Velocity (32-bit). This is the raw speed value (0 at rest, grows with throttle).

### 0x0604D6B8 writes:
- +0x34: Speed display value (at pc=0x0604D70A). This is the km/h integer shown on-screen.
  Formula: velocity * 0x006C0000 → DMULS → XTRCT → SHLR16 → clamp to [0, 0x14E]
  Then clamp to [0, 0x2AAA] (secondary cap), add to current +0x34 value.

### Constants used:
- 0x000000B8 — upper clamp for input state
- 0x00000038 — lower clamp / idle value for input state
- 0x006C0000 — velocity-to-speed multiplier
- 0x0000014E — speed clamp (334 decimal = max ~334 km/h?)
- 0x00002AAA — secondary speed clamp
- 0x00000140 — threshold for flag set at +0x30
- 0x00000080 — used as FUN_06048160 parameter

## Other Observations

- FUN_0604D580 and its continuations (0x0604D6B8, 0x0604D658, 0x0604D758, 0x0604D780)
  form a coherent input-processing and speed-calculation pipeline.
- The code flow: raw input → clamped/scaled → physics state updates → display value.
- FUN_06048160 (at 0x06048160) is called as a math helper (appears to be a
  divide or fixed-point scale routine). Called via r12. Used in both steering
  and speed calculation paths.
- The `.4byte sym_046E0626` at line 392 is an encoded instruction in the byte fog,
  likely `mov.l r6, @(r0, r4)` — the GBR+0x7C write instruction.
- The speed clamp of 0x14E (334) suggests max possible displayed speed is 334 km/h.
- The +0x88/+0x98 fields use a range of [0x38, 0xB8] — 0x38 is idle, 0xB8 is full input.
  The delta (0x80 = 128) is then scaled by 0xFF and divided to produce the +0x8C/+0x9C
  outputs.
